§ 01

Introduction & History

Why C exists and why it still matters

C was created by Dennis Ritchie at Bell Labs between 1969 and 1973, primarily for writing the UNIX operating system. It evolved from B, which itself derived from BCPL. Today, decades later, C remains the lingua franca of systems programming.

<1>

Portable

Write once, compile on any architecture — x86, ARM, RISC-V, MIPS.

<2>

Close to Hardware

Direct memory access, pointer arithmetic, bitwise ops — full control.

<3>

Minimal Runtime

No garbage collector, no virtual machine — deterministic behavior.

<4>

Influence

C, C++, Java, Python runtime, Go, Rust — all trace heritage to C.

ℹ
C Standards: K&R C (1978) → C89/ANSI → C99 → C11 → C17 → C23. Each revision adds features while preserving backward compatibility. Prefer -std=c11 or -std=c17 for modern code.

Basic Syntax & Structure

Anatomy of a C program

#include <stdio.h> /* standard I/O library */ /* Every C program starts here */ int main(void) { printf("Hello, World!\n"); return 0; /* 0 = success */ }

#include pulls in header files before compilation begins.

{ }

Blocks

Curly braces delimit scope. Every function body lives in a block.

;

Statements

Each statement ends with a semicolon. Missing one = compile error.

main

Entry Point

main() is mandatory. OS calls it when the program starts.

Comments use /* block */ or // line (C99+). They are stripped by the preprocessor.

Identifiers start with a letter or underscore, followed by letters, digits, or underscores. Case-sensitive: myVar ≠ myvar.

§ 06

Conditionals

if/else, switch, and short-circuit evaluation

conditionals.cC
/* if / else if / else */ if (score >= 90) { puts("A"); } else if (score >= 80) { puts("B"); } else { puts("C or below"); } /* switch: integral types only */ switch (day) { case 1: puts("Monday"); break; case 6: case 7: puts("Weekend"); break; default: puts("Midweek"); break; } /* Short-circuit: safe null check */ if (ptr != NULL && ptr->value > 0) { process(ptr); }
!
Fallthrough: Without break, execution falls through to the next case. This is intentional sometimes (grouping cases) but a common source of bugs. Use [[fallthrough]] (C23) or a comment to signal intentional fallthrough.

Functions

Declaration, definition, scope, and call by value

functions.cC
#include <stdio.h> #include <math.h> /* Forward declaration (prototype) */ double hypotenuse(double a, double b); int main(void) { printf("%.2f\n", hypotenuse(3.0, 4.0)); /* 5.00 */ return 0; } /* Definition */ double hypotenuse(double a, double b) { return sqrt(a*a + b*b); } /* Void function */ void swap(int *a, int *b) { /* pass by pointer */ int tmp = *a; *a = *b; *b = tmp; } /* Variadic function (like printf) */ #include <stdarg.h> int sum(int count, ...) { va_list args; va_start(args, count); int total = 0; for (int i = 0; i < count; i++) total += va_arg(args, int); va_end(args); return total; }
ℹ
Call by value: C passes all arguments by value (copies). To modify the caller's variable, pass a pointer. Arrays decay to a pointer to their first element automatically.

Recursion

Stack frames, base cases, and tail recursion

recursion.cC
/* Factorial — classic recursion */ long long factorial(int n) { if (n <= 1) return 1; /* base case */ return n * factorial(n - 1); } /* Fibonacci with memoization */ long long memo[100] = {0}; long long fib(int n) { if (n <= 1) return n; if (memo[n]) return memo[n]; return memo[n] = fib(n-1) + fib(n-2); } /* Tail-recursive version (GCC optimizes to a loop) */ long long fact_tail(int n, long long acc) { if (n <= 1) return acc; return fact_tail(n - 1, n * acc); }
§ 10

Pointers

The heart of C — addressing, dereferencing, pointer arithmetic

A pointer is a variable whose value is a memory address. Pointers give C its power — direct memory access, efficient data structures, and the ability to write to caller variables.

// pointer relationship in memory
42
int x = 42
addr: 0x7ffe0004
→
0x7ffe0004
int *p = &x
addr: 0x7ffe0008
*p == 42 (dereference)
&x == p (address-of)
p == 0x7ffe0004
pointers.cC
int x = 42; int *p = &x; /* & = address-of */ int val = *p; /* * = dereference → 42 */ *p = 100; /* modifies x through pointer */ /* Pointer arithmetic */ int arr[] = {10, 20, 30}; int *q = arr; printf("%d\n", *(q + 1)); /* 20 */ printf("%d\n", q[2]); /* 30 — same as *(q+2) */ /* NULL pointer — never dereference */ int *np = NULL; if (np != NULL) *np = 5; /* Pointer to pointer */ int **pp = &p; printf("%d\n", **pp); /* 100 */ /* const pointers */ const int *cp = &x; /* can't modify *cp */ int *const pc = &x; /* can't change address */
✕
Dangling pointers: never use a pointer after the memory it points to has been freed or has gone out of scope. Use p = NULL immediately after free(p) to catch use-after-free bugs.

§ 12

Strings

Null-terminated char arrays and <string.h>

C strings are null-terminated arrays of char. The terminator '\0' (ASCII 0) marks the end. Always allocate space for it.

strings.cC
#include <string.h> char s1[] = "Hello"; /* 6 bytes: H e l l o \0 */ char s2[32]; const char *lit = "literal"; /* string literal — read only */ strlen(s1); /* 5 (excludes \0) */ strcpy(s2, s1); /* copy */ strcat(s2, ", world"); /* concatenate */ strcmp(s1, s2); /* 0 if equal */ strncpy(s2, s1, 31); /* safe: limit copy length */ snprintf(s2, sizeof(s2), "%s%d", "x", 42); /* Character classification <ctype.h> */ isalpha('a'); isdigit('9'); toupper('a'); /* Manual iteration */ for (char *c = s1; *c != '\0'; c++) putchar(*c);
✕
Buffer overflow is a critical security vulnerability. Prefer strncpy, strncat, snprintf, and fgets over their unbounded counterparts (strcpy, strcat, gets). Never use gets() — it has no length limit.

Heap allocation — malloc, calloc, realloc, free

// <stdlib.h> memory functions
malloc(n)
Allocate n bytes — uninitialized

Allocate n × sz bytes — zero-initialized

Release heap memory
heap.cC
#include <stdlib.h> int n = 100; /* malloc — always check for NULL */ int *arr = malloc(n * sizeof(int)); if (!arr) { perror("malloc"); return 1; } /* calloc — zeroed memory */ double *mat = calloc(10, sizeof(double)); /* realloc — may return new pointer */ int *tmp = realloc(arr, n * 2 * sizeof(int)); if (tmp) arr = tmp; /* don't clobber if realloc fails */ /* Always free — prevent memory leaks */ free(arr); arr = NULL; free(mat); mat = NULL;
✓
Valgrind and AddressSanitizer are essential tools. Compile with -fsanitize=address,leak during development to detect leaks, use-after-free, and buffer overruns automatically.

Linked Lists

Singly, doubly, and circular — with full implementation

linked_list.cC
typedef struct Node { int data; struct Node *next; } Node; /* Prepend O(1) */ Node *prepend(Node *head, int val) { Node *n = malloc(sizeof(Node)); n->data = val; n->next = head; return n; } /* Append O(n) */ Node *append(Node *head, int val) { Node *n = malloc(sizeof(Node)); n->data = val; n->next = NULL; if (!head) return n; Node *cur = head; while (cur->next) cur = cur->next; cur->next = n; return head; } /* Delete and free all */ void free_list(Node *head) { Node *tmp; while (head) { tmp = head->next; free(head); head = tmp; } } /* Reverse in-place */ Node *reverse(Node *head) { Node *prev = NULL, *cur = head, *nxt; while (cur) { nxt = cur->next; cur->next = prev; prev = cur; cur = nxt; } return prev; }

Stacks & Queues

Array and linked-list implementations

stack_queue.cC
/* ── STACK (array-backed) ── */ #define MAX 256 typedef struct { int data[MAX]; int top; } Stack; void stack_push(Stack *s, int v) { s->data[++s->top] = v; } int stack_pop (Stack *s) { return s->data[s->top--]; } int stack_peek(Stack *s) { return s->data[s->top]; } int stack_empty(Stack *s) { return s->top == -1; } /* ── QUEUE (circular array) ── */ typedef struct { int data[MAX], front, rear, size; } Queue; void enqueue(Queue *q, int v) { q->data[q->rear] = v; q->rear = (q->rear + 1) % MAX; q->size++; } int dequeue(Queue *q) { int v = q->data[q->front]; q->front = (q->front + 1) % MAX; q->size--; return v; }

Preprocessor

Macros, conditional compilation, include guards

preproc.cC
/* ── Object-like macros ── */ #define PI 3.14159265358979 #define MAX 1024 /* ── Function-like macros ── */ #define SQ(x) ((x) * (x)) /* always parenthesize! */ #define MAX2(a, b) ((a) > (b) ? (a) : (b)) /* ── Variadic macro (C99) ── */ #define LOG(fmt, ...) fprintf(stderr, fmt, __VA_ARGS__) /* ── Stringification & concatenation ── */ #define STR(x) #x #define CAT(a,b) a##b /* ── Include guard (or #pragma once) ── */ #ifndef MYHEADER_H #define MYHEADER_H /* header contents */ #endif /* ── Conditional compilation ── */ #ifdef DEBUG #define ASSERT(x) do { if(!(x)) abort(); } while(0) #else #define ASSERT(x) ((void)0) #endif /* ── Predefined macros ── */ // __FILE__ __LINE__ __func__ __DATE__ __TIME__
!
Macro pitfalls: macros lack type checking and scoping. Prefer inline functions and const variables (C99+) for type safety. Always wrap macro arguments in parentheses to avoid operator-precedence bugs.

§ 23

Memory Layout

Stack, heap, text, BSS — the full picture

// Linux process memory map (high → low)
Stack local vars, return addresses — grows ↓
↕ (ASLR gap)
Heap malloc/free — grows ↑
BSS uninitialized globals (zero-filled by OS)
Data initialized globals & statics
Text executable code (read-only)

Stack stores automatic variables and return addresses. It is fixed-size (default ~8 MB on Linux). Heap is managed manually via malloc/free. BSS/Data hold global and static variables. Understanding this layout is essential for debugging, security, and performance.
